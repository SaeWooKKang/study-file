# TS

## TS와 JS의 관계
- TS는 JS의 상위집합(superset)이다.
- 모든 JS는 TS이다. 

## TS의 역할
- TS를 JS로 변환하는 역할(컴파일)
  - TS의 컴파일 역할
    - 트랜스파일
      - 컴파일시 타입은 제거됨
        - 런타임에 타입 체크 불가능 
    - 타입 오류 체크
    - 컴파일과 타입체크는 독립적으로 동작
      - 타입 오류가 있는 코드도 컴파일 가능 
- 타입 시스템 
  - 목표
    - 런타임에 오류를 발생시킬 코드를 미리 찾아내는 것 
    - ( 의도와 다른 코드까지 찾아줌 )
  - TS설치시 tsserver 명령어 실행 가능
    - 언어 서비스 제공(편집기를 통해)
  - JS의 렌더링 동작을 모델링함
    - JS는 덕 타입핑 기반이고 TS도 무엇으로 선언되었는지는 필요없고, 요구사항만 만족하면 됨
